Admininstrator

The key tech of this project is Javaagent. It is loaded before other classes when the JVM started. So that we can use it to modify the classes we want to monitor. Right now, we add one line at every methods in these classes. Then when these methods are called, our backend will receive the notification. With Javaagent, the major issue is performance. We use multithread and async way to solve. When we want to send message to server, we do not directly send it, but put it to queues, then use threads to handle them when it is not so stress. Below is the example settings for WebSphere, you need to change the setting file at: C:\IBM\WebSphere\AppServer\profiles\AppSrv01\config\cells\localhostCell\nodes\localhost\servers\CuramServer\server.xml and change below setting to:

genericJvmArguments="-javaagent:C:\IBM\WebSphere\AppServer\profiles\AppSrv01\installedApps\localhostCell\Curam.ear\Agent.jar=focusOn[facade],hostOn[jiens-mbp.mul.ie.ibm.com:8080],team[devOps],version[7.0.3.0],build[1234]"

Edit Test Cases

MongoDB is our backend Database, it is faster and no restriction on data structure. Of course, it does not support transaction; but we don't really care about this. We use Cache (JCS: Java Cache System) to store the recent query results in the memory, so that the pressure on the Database server will be lower and the response to front end will be really fast. The only interface for the backend is REST API.
